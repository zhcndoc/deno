List of rules

These lint rules are provided by the deno lint command. You can enable sets of rules in deno.json(c) by adding their tags (e.g. recommended, react) to the lint.rules.tags array.

If no tag is provided, then the recommended set of rules will be enabled by default.

  •  Recommended
  •  Fresh
  •  JSX
  •  React
  •  JSR
  •  Workspace
  • adjacent-overload-signatures

    要求重载签名必须相邻。 Details about undefined

  • ban-ts-comment

    不允许在没有注释的情况下使用 Typescript 指令。 Details about undefined

  • ban-types

    禁止使用原始包装对象（例如 String 这个对象是 string 原始类型的包装），以及不明确的 Function 类型和误解的 Object 类型。 Details about undefined

  • ban-unknown-rule-code

    警告在忽略指令中使用未知规则代码。 Details about undefined

  • ban-untagged-ignore

    需要将 deno-lint-ignore 注解与一个或多个规则名称。 Details about undefined

  • ban-untagged-todo

    要求 TODO 注释必须带有用户标签（@user）或问题引用（#issue）。 Details about undefined

  • ban-unused-ignore

    警告未使用的忽略指令。 Details about undefined

  • camelcase

    强制在变量名中使用 camelCase。 Details about undefined

  • constructor-super

    验证构造函数和 super() 调用的正确使用。 Details about undefined

  • default-param-last

    强制默认参数在函数签名中位于最后。 Details about undefined

  • eqeqeq

    强制使用类型安全的相等运算符 === 和 !== 代替更容易出错的 == 和 != 运算符。 Details about undefined

  • explicit-function-return-type

    要求所有函数都有明确的返回类型。 Details about undefined

  • explicit-module-boundary-types

    要求所有模块导出具备完全类型声明。 Details about undefined

  • for-direction

    需要 for 循环控制变量以正确的方向递增。 Details about undefined

  • fresh-handler-export

    检查命名是否正确，适用于命名的 fresh 中间件导出。 Details about undefined

  • fresh-server-event-handlers

    不允许在 fresh 服务器组件中使用事件处理程序。 Details about undefined

  • getter-return

    要求所有属性获取器函数返回一个值。 Details about undefined

  • guard-for-in

    要求 for-in 循环包含 if 语句。 Details about undefined

  • jsx-boolean-value

    强制一致的 JSX 布尔值样式。传递 true 作为布尔值可以使用简写语法省略。 Details about undefined

  • jsx-button-has-type

    Enforce <button> elements to have a type attribute. If a <button> is placed inside a <form> element it will act as a submit button by default which can be unexpected. Details about undefined

  • jsx-curly-braces

    确保在 JSX 表达式周围一致使用花括号。 Details about undefined

  • jsx-key

    确保在将可迭代对象传递给 JSX 时存在 key 属性。这允许框架优化元素顺序的检查。 Details about undefined

  • jsx-no-children-prop

    将子元素作为 JSX 子元素传递，而不是作为属性。 Details about undefined

  • jsx-no-comment-text-nodes

    JavaScript comments inside text nodes are rendered as plain text in JSX. This is often unexpected. Details about undefined

  • jsx-no-duplicate-props

    不允许重复的 JSX 属性。后面的属性将始终覆盖前面的属性，这常常会导致意想不到的结果。 Details about undefined

  • jsx-no-unescaped-entities

    Leaving the > or } character in JSX is often undesired and difficult to spot. Enforce that these characters must be passed as strings. Details about undefined

  • jsx-no-useless-fragment

    片段仅在 JSX "块" 的顶部是必要的，并且仅在有多个子元素时需要。在其他情况下不需要片段。 Details about undefined

  • jsx-props-no-spread-multi

    重复同样的表达式通常是一个错误，并且会导致不必要的计算。 Details about undefined

  • jsx-void-dom-elements-no-children

    确保 HTML 中的空元素没有任何子元素，因为这不是有效的 HTML。有关更多信息，请参阅 空元素 文章在 MDN。 Details about undefined

  • no-array-constructor

    强制执行数组构造的常规用法。 Details about undefined

  • no-async-promise-executor

    要求不使用 async promise 执行器函数。 Details about undefined

  • no-await-in-loop

    在 for 循环体中未使用 await。 Details about undefined

  • no-await-in-sync-fn

    禁止在非异步函数中使用 await 关键字。 Details about undefined

  • no-boolean-literal-for-arguments

    要求所有函数在调用时，无论参数数量为多少，都使用自文档常量代替 boolean 字面量作为参数。 Details about undefined

  • no-case-declarations
