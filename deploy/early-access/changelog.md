## 2025年6月12日

### 功能

- Deno Deploy<sup>EA</sup> 现已支持 playgrounds！
  - playgrounds 可以通过组织概览中的 playgrounds 标签页创建和访问。
  - playgrounds 可以包含多个文件，并支持构建步骤。
  - playground UI 中包含一个 iframe，用于预览已部署的应用。
  - 目前提供三种模板：hello world、Next.js 和 Hono。
- 移动端新增悬浮导航栏，不再遮挡页面内容。

## 2025年6月9日

### 功能

- Deno Deploy<sup>EA</sup> 有了新的标志！
- 现在任何人都可以通过注册 [dash.deno.com](https://dash.deno.com/account#early-access) 来加入早期访问。
- 构建
  - 构建可用存储从 2 GB 提升至 8 GB。
  - 构建现可使用在组织或应用设置中配置的环境变量和密钥（位于新的“构建”上下文中）。
  - 构建最大运行时间现在为 5 分钟。
- 指标页面经过全面重写图表渲染：
  - 可通过拖动图表放大选中区域。
  - 支持显示更多数据而不会导致页面加载变慢。
  - 工具提示跟随鼠标光标，同时新增十字线便于精确分析。
  - 字体大小和颜色提升了可读性。

### Bug 修复

- 构建不再卡在待处理状态。
- 仪表盘页面加载速度显著提升。
- 追踪中正确显示未导出的父跨度。
- 指标页面切换时间范围时正确刷新。
- 遥测搜索栏中的“清除搜索”按钮现可正常工作。
- 较旧版本的 Next.js（如 Next.js 13）现可正确构建。
- 环境变量抽屉现统一使用，修复多个同名但不同上下文环境变量冲突的问题。
- 在构建器中运行 `node <path>` 时，路径为绝对路径不会再失败。
- 构建器中现已支持 `npx`。
- Astro 构建不再随机出现 `--unstable-vsock` 错误。
- 当项目显式指定 `@deno/svelte-adapter` 时，Svelte 项目能正确部署。

## 2025年5月26日

### 功能

- 触发手动构建时，现在可以选择部署哪个分支。
- 现可部署 Astro 静态网站，无需手动安装 Deno 适配器。
- 提供了 [供您浏览的参考文档](https://docs.deno.com/deploy/early-access/)。

### Bug 修复

- 使用 `npm` 作为包管理器时，SvelteKit 自动检测功能现已正常工作。
- 预热功能不再随机触发对您的应用的 POST 请求。
- 访问带尾部斜杠的页面不再返回 404。
- 在抽屉内部点击、按住并拖动背景再释放时，抽屉不再关闭。

## 2025年5月22日

### 功能

- 现在可以通过粘贴 `.env` 文件的方式批量导入环境变量，方便创建应用。
- SvelteKit 现开箱即用，无需手动安装 Deno 适配器。
- 已提供 Lume 静态网站生成器的预设配置。

### Bug 修复

- 环境变量现可在时间线页面正确显示。
- 生产时间线页面能够正确显示所有构建。
- app.deno.com 现支持较旧版本的 Firefox。
- app.deno.com 上的页面标题现正确反映当前页面。
- “申请证书”按钮在 DNS 验证失败后不再卡死。
- 之前有证书或已关联应用的域名现可删除。
